  • PostgresSQL

Recipe Box is your Modern day Recipe Book, it can help even the most seasoned Cooks to very easily store and manage their Recipes on the fly while also enabling the culinary curious user to browse thousand of new recipes, via search by Dish/ingredient, or Course.

During this project we encountered a problem with displaying ingredients as a list since they were stored as a string in SQL. I created a function that turns the stored string into an array of ingredients and then use .forEach function to turn it into an object. With the object now I can use template literals to display ingredients as a list

Knitworthy uses the Ravelry API to pull information about users' projects and compare project data to real life time and length measurements to put the amount of love that goes into a handmade gift into perspective for someone who does not knit or crochet.

The challenging aspect of this project is that Ravelry API is not completely filled. Users leave a lot of information blank and this creates errors when we try to display them. After we get the data from fetching API, we run it through a function that checks for all null values and replace them.

Event Master

  • Javascript
  • NodeJs
  • Express
  • React
  • Redux
  • PostgresSQL
  • Sequelize
  • RESTFUL-API

Event master is a event sharing apps. At its basic, Event master is about highly customizable rapid event invitation creating. In addition to normal event creation, this app allows users to notify their guests about any changes, updates, or warning about covid exposure.

The project's main obstacle is to display events in chronological order because the date and time are stored seperately. To overcome this problem, I combine date time data into one integer (year-month-date-hours-minutes-seconds) and sort the events based on this.
